## 定義

エージェントノードはDify Chatflow/Workflowのコンポーネントで、自律的なツール呼び出しを可能にします。さまざまなエージェント推論戦略を統合することで、LLMはランタイムにツールを動的に選択し実行し、複数ステップの推論を実行できます。

## 設定手順

### ノードの追加

Dify Chatflow/Workflowエディターで、コンポーネントパネルからエージェントノードをキャンバスにドラッグします。

![](https://assets-docs.dify.ai/dify-enterprise-mintlify/ja-jp/guides/workflow/node/1f4d803ff68394d507abd3bcc13ba0f3.png)

### エージェント戦略の選択

ノード設定パネルで、エージェント戦略をクリックします。

![](https://assets-docs.dify.ai/dify-enterprise-mintlify/ja-jp/guides/workflow/node/f14082c44462ac03955e41d66ffd4cca.png)

ドロップダウンメニューから、希望のエージェント推論戦略を選択します。Difyは**関数呼び出しとReAct**の2つの組み込み戦略を提供しており、これらは**Marketplace → Agent Strategiesカテゴリー**からインストールできます。

![](https://assets-docs.dify.ai/dify-enterprise-mintlify/ja-jp/guides/workflow/node/47e29e5993821b61632af9cdb8392357.png)

#### 1. 関数呼び出し

関数呼び出しは、ユーザーのコマンドを事前に定義された関数やツールにマッピングします。LLMは最初にユーザーの意図を特定し、その後、どの関数を呼び出し、必要なパラメータを抜き出すかを決定します。その核心的なメカニズムは、外部関数やツールを明示的に呼び出すことです。

長所：

**• 正確さ:** 明確に定義されたタスクの場合、複雑な推論を必要とせずに対応するツールを直接呼び出すことができます。

**• 外部機能の統合が容易:** さまざまな外部APIやツールを関数にラップして、モデルが呼び出せます。

**• 構造化された出力:** モデルは機能呼び出しに関する構造化された情報を出力し、ダウンストリームノードによる処理を促進します。

![](https://assets-docs.dify.ai/dify-enterprise-mintlify/ja-jp/guides/workflow/node/10505cd7c6f0b3ba10161abb88d9e36b.png)

#### 2. ReAct (Reason + Act)

ReActは、エージェントが推論と行動を交互に行うことを可能にします：LLMは最初に現在の状態と目標を考慮し、その後、適切なツールを選択し呼び出します。ツールの出力は、次の推論と行動のステップに情報を提供します。このサイクルは問題が解決されるまで続きます。

長所：

**• 外部情報の効果的な利用:** モデル単独では達成できないタスクを処理し、情報を取得するために外部ツールを活用できます。

**• 説明性の向上:** 推論と行動が織り交ぜられているため、エージェントの思考過程にはある程度の追跡可能性があります。

**• 幅広い適用性:** Q\&A、情報の取得、タスクの実行など、外部知識が必要なシナリオや特定の行動を実行する必要があるシナリオに適しています。

![](https://assets-docs.dify.ai/dify-enterprise-mintlify/ja-jp/guides/workflow/node/60fa430029e509ac1a609c72fd04c413.png)

開発者はエージェント戦略プラグインを公開リポジトリに寄与できます。審査後、これらのプラグインはMarketplaceにリストされ、他の人がインストールすることができます。

### ノードパラメータの設定

エージェント戦略を選択した後、設定パネルに関連するオプションが表示されます。Difyと共に出荷される関数呼び出しとReAct戦略では、利用可能な設定項目は以下を含みます：

1. **モデル:** エージェントを駆動する大規模な言語モデルを選択します。
2. **ツールリスト:** ツールの使用方法はエージェント戦略によって定義されます。+をクリックして、エージェントが呼び出すツールを追加および設定してください。
   * 検索: ドロップダウンからインストール済みのツールプラグインを選択します。
   * 認証: ツールを有効にするためにAPIキーやその他の認証情報を提供します。
   * ツールの説明とパラメータ設定: LLMがツールを使用する理由と時期を理解するのに役立つ説明を提供し、機能パラメータを設定します。
3. **指示**: エージェントのタスク目標とコンテキストを定義します。上流ノード変数を参照するためにJinja構文がサポートされています。
4. **クエリ**: ユーザーの入力を受け取ります。
5. **最大反復回数:** エージェントの実行ステップの最大数を設定します。
6. **出力変数:** ノードによって出力されるデータ構造を示します。

![](https://assets-docs.dify.ai/dify-enterprise-mintlify/ja-jp/guides/workflow/node/54c8e4f0eaa7379bd8c1b5ac6305b326.png)

## ログ

実行中、エージェントノードは詳細なログを生成します。ノード実行全体の情報（入力と出力、トークン使用量、所要時間、ステータス）が表示されます。エージェント戦略実行の各ラウンドからの出力を表示するために「詳細」をクリックします。

![](https://assets-docs.dify.ai/dify-enterprise-mintlify/ja-jp/guides/workflow/node/0120d3d0e63f5b59ec9d279a38c970ef.png)

## メモリ

**メモリ**トグルを有効にすると、エージェントが会話のコンテキストを保持し、思い出すことが可能になります。**ウィンドウサイズ**を調整することで、エージェントが「記憶」できる以前の会話メッセージの数を制御できます。これにより、エージェントは以前の交流を理解し、参照することができ、多ターンの対話体験を向上させる一貫性のある文脈に関連する応答を提供します。

たとえば、ユーザーが後続のメッセージで代名詞（「それ」、「これ」、「彼ら」など）を使用する場合、メモリが有効なエージェントは、ユーザーが完全な情報を再度述べる必要なしに、以前のコンテキストからこれらの代名詞が何を指しているかを理解できます。

### ツールの説明

MCPサーバーから受け取るデフォルトの説明を上書きすることができます。これは、説明を特定のユースケースにより具体的にするのに役立ちます。

### パラメータ設定（推論コンフィグ）

各ツールパラメータについて、次のような選択が可能です：

**自動**: AIモデルがコンテキストに基づいてパラメータ値を決定する（デフォルトの動作）

**固定値**: 常に使用され、AI推論からパラメータを取り除く特定の値（静的値または変数）を設定します。

これは次にのために強力です：

- 検索ツール用に `numResults: 10` のような一貫した設定値を設定する
- 変更すべきでないパラメータ（特定のAPIエンドポイントやフォーマット設定）の事前充填を設定する
- AIが扱う必要があるパラメータの数を減らすことでツールの使いやすさを簡素化します。

例えば、ウェブ検索ツールでは：

- AIが検索する内容を決定するために「自動」で`query`を保持する
- 応答サイズを制限するために、`numResults`を「5」のような固定値に設定します
- 一貫した動作のために検索フィルタのような他のパラメータを固定値に設定します。
